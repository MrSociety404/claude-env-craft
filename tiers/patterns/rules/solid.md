---
name: SOLID Principles
description: Apply SOLID design principles for maintainable architecture
tier: patterns
---

- **Single Responsibility**: Each class/module has one reason to change
- **Open/Closed**: Extend behavior via composition or inheritance, not modification of existing code
- **Liskov Substitution**: Subtypes must be substitutable for their base types without breaking behavior
- **Interface Segregation**: Prefer many small, focused interfaces over one large interface
- **Dependency Inversion**: High-level modules depend on abstractions, not concrete implementations
