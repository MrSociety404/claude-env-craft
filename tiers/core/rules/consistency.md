---
name: Code Consistency
description: Maintain consistent style and patterns within the project
tier: core
---

- Follow existing patterns in the codebase — don't introduce new paradigms without discussion
- Use the same approach for similar problems across the project
- Keep formatting consistent: 2 spaces indentation, consistent quote style
- Import ordering: external packages first, then internal modules, then relative imports
- File structure should follow a consistent pattern within its category
- When adding code, match the style of surrounding code
