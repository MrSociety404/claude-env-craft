---
name: Internationalization
description: i18n patterns for @nuxt/i18n
paths: ["**/*.vue", "i18n/**", "locales/**"]
---

- Never hardcode user-facing strings — always use `$t('key')` or `t('key')`
- Translation keys use dot notation hierarchy: `pages.home.title`, `components.navbar.logout`
- Locale files organized by feature/page, not as one giant file
- Default locale file is the source of truth — other locales mirror its structure
- Use ICU message format for plurals and interpolation
- Date/number formatting through `$d()` and `$n()`, not manual formatting
- Route localization: use `useLocalePath()` for internal links
- New components/pages must include translation keys before merge
